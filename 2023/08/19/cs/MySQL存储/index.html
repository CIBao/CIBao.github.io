# MySQL 的数据存放在哪个文件？

InnoDB 默认引擎

表 t_order 在 /var/lib/mysql/db/（默认三个文件）

db.opt：字符集和检验规则

t_order.frm：表结构

t_order.ibd：表数据（默认独占表）

  • 行：记录按行存放

  • 页：按页读取（16KB）（数据页、日志页、溢出页）

  • 段：连续物理空间（1MB、64 页、减少随机 IO）

  • 区：

    • 索引段：非叶子节点区
    • 数据段：叶子节点区
    • 回滚段：回滚数据区

# InnoDB 行格式（Compact）

变长字段长度列表

为什么「变长字段长度列表」「NULL」的信息要按照逆序存放？指针位置、cache

每个数据库表的行格式都有「变长字段字节数列表」吗？ NO、省空间

NULL 值列表

每个数据库表的行格式都有「NULL 值列表」吗？NO、省空间（至少 1 字节）

记录头信息

  • delete_mask ：标识此条数据是否被删除。
  • next_record：下一条记录的位置
  • record_type：类型（0 普通、1 非叶、2 最小、3 最大）

row_id：隐藏字段（没有主键时）、6B

trx_id: 哪个事务生成、6B

roll_pointer：上一个版本的指针（必需）、7B
