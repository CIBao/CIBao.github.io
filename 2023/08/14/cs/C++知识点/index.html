# C++11

# 1. 新特性

nullptr
auto decltype (类型推导)
for(auto)
Lambda
单向链表
右值引用和 move

# 2、auto、decltype 和 decltype (auto) 的用法

auto：编译器分析表达式类型（必须初始化）
decltype: 返回操作的类型，不会初始化
decltype (auto)： auto 会被等式右边替换

# 3、C++ 中 NULL 和 nullptr 区别

C：NULL == void*（0）
CPP：NULL == 0（nullptr 为了区分指针和整数 0）

# 4、智能指针的原理、常用的智能指针及实现

原理：将普通指针（指向动态分配对象）封装成一个类。
负责自动释放（当类对象生命周期结束时，自动调用析构函数释放资源）

共享指针：引用计数，当计数为 0 的时候，自动调用析构函数
线程安全的，对象的读取是要上锁的
独占指针：不允许拷贝，一个对象只能由一个独占指针指向
弱指针：解决共享指针中的互相引用，本质是只引用不计数
auto 指针：解决异常抛出时内存泄漏

# 5、说一说你了解的关于 lambda 函数的全部知识

作用：编写内嵌的匿名函数，代替独立函数或函数对象
原理：编译器生成匿名类（重载（）运算符），又称闭包，运行时，会返回匿名的闭包实例（右值）

# 1、类的对象存储空间？

对齐的 padding

空类：1 （作为基类：0）

# 2、简要说明 C++ 的内存分区

栈（小）、堆（大）、全局 / 静态（初始化为 0）、常量区、代码区

# 6、关于 this 指针你知道什么？全说出来

类的指针：指向对象首地址
只能调用成员函数（只有在这有定义）

在非静态函数访问非静态成员，编译器会隐含传递 this 指针

场景：return *this 和 用于区分同名

this 在成员函数的开始执行前构造，在成员的执行结束后清除

this 在哪根据编译器决定，可能是栈，也可能是寄存器，甚至全局变量

# 8、内存泄漏的后果？如何监测？解决方法？

内存泄漏：未释放不用的内存
后果：内存不足 OOM
检测：BoundsChecker CRT (C run-time libraries)
方法：智能指针

# 9、在成员函数中调用 delete this 会出现什么问题？对象还可以使用吗？

对象所占据的内存释放（数据成员和 this）, 能访问不涉及 this 的函数，
但一旦涉及 this 就会出现不可预期的问题（内存不是立马回收到 OS，
大概会出现可能访问得了，但是值不确定，虚函数表的指针无效）
