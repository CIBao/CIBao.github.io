# 1. 什么是 STL？

算法。容器。迭代器

# 2.trivial destructor

未定义的析构函数

系统判断是不是 trivial
是就什么都不做
不是就调用 destroy ()

# 3.RAII

资源获取即初始化，构造函数中申请，析构函数中释放

例子：智能指针

# 4. ➕➕it、it++ 哪个好

前者返回引用
后者返回对象（临时对象，效率低）

# 5.C++ 左值引用和右值引用

左值：有名字、取地址
右值：反之

左值引用：只能引用左值，但是 const 的话左右都行
右值引用：只能引用右值，通过移动语义避免多次拷贝，让临时变量生存期变长

例子：x = 复制构造函数 (), 返回时创建临时变量，在调用拷贝函数，临时变量销毁
而用右值应用时，返回时临时变量，临时变量不会销毁。

# 8.STL 的两级空间配置器

原因：频繁申请释放会产生大量外部碎片

一级配置器：大于 128KB，allocate.deallocate.reallocate

二级配置器：小于 128KB，16 条空闲链表（8-128KB）
free 空时：从内存池申请 20 个节点大小，分配 1 个给用户，剩余 19 个当 free
内存池不够：则用 malloc 分配

# 9.vector 与 list 的区别与应用？

vector 连续内存空间，动态增长
list 双向链表

# 10.STL 中 vector 删除其中的元素，迭代器如何变化？为什么是两倍扩容？释放空间？

size () 已用空间
capacity () 总空间大小
当 size () == capacity () 会引起扩容。

reverse (n) > capacity 才生效
resize (n) 只改变元素个数

装填因子在 50% 以上，不浪费，太小要经常扩容

# 11.Vector 如何释放空间？

只增不减。
用 deque 或调用 swap () 可以释放空间

# 12. 容器内部删除一个元素

顺序容器: it = c.erase (it)
关联容器: c.erase (it++)

# 13.STL 迭代器如何实现

迭代器：不用了解容器内部，遍历容器
实现：容器相关联的指针，对指针进行封装，重载各种运算符

# 14.map.set 是怎么实现的，红黑树是怎么能够同时实现这两种容器？ 为什么使用红黑树？

实现：红黑树（自动排序），插入删除 Logn
模板参数：key（set， map），set 节点（value），map 节点（key+value）

区别：有序和无序
原理：hashtable 用 hash 判断（重载 ==），红黑树用值判断（重载 <）

大于阈值（加载因子 * 数组长度）扩容

# 19.map 中 [] 与 find 的区别？

[]: 作为下标去找返回值，若不存在则插入 key 和默认值
find: 返回迭代器，若不在则返回尾迭代器

# 20.STL 中 list 与 queue 之间的区别

deque: 连续空间，随时增加新的空间连接

# 21.STL 中的 allocator.deallocator

allocator: 大于 128KB，调用一级配置器，找到 freelist 释放，若没有则（同 #8）
deallocator：大于 128KB，调用一级配置器，小于则找到对应 freelist 释放
