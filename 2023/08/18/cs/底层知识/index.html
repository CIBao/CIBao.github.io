# 红黑树

红黑树（Red-Black Tree）是一种自平衡的二叉搜索树，它的每个节点都有一个额外的表示节点颜色的位，可以是红色或黑色。

  1. 每个节点要么是红色，要么是黑色。
  2. 根节点是黑色。
  3. 每个叶子节点（NIL 节点，即空节点）都是黑色。
  4. 如果一个节点是红色的，则它的两个子节点都是黑色的。
  5. 对于每个节点，从该节点到其所有子孙叶子节点的简单路径上，均包含相同数目的黑色节点。

通过满足这些特性，红黑树能够保持树的高度相对较低，从而保证了树的查找、插入和删除操作的时间复杂度为 O (log n)。

红黑树的自平衡性是通过节点的旋转和重新着色来实现的。当插入或删除节点导致红黑树失衡时，可以通过一系列的旋转和重新着色操作来重新平衡树。这些操作确保了红黑树始终保持平衡状态。

红黑树广泛应用于实现频繁的查找、插入和删除操作的场景中。（CFS 调度器）

# 为什么不选 AVL 树原因：

TL，DR：红黑树插入删除代价更少。

红黑树和 AVL 树都是自平衡的二叉搜索树，它们都能够保持树的高度相对较低，从而保证了树的查找、插入和删除操作的时间复杂度为 O (log n)。

  1. 平衡性维护成本：红黑树相对于 AVL 树来说，旋转和重新着色的操作更少，更简单。AVL 树需要在每次插入或删除节点时进行更多的旋转操作来保持平衡，而红黑树只需要进行最多两次旋转和重新着色。这使得红黑树的维护成本更低。
  2. 内存占用：AVL 树需要为每个节点额外存储一个平衡因子（balance factor）来表示节点的平衡状态，通常需要一个整数来表示。而红黑树只需要一个二进制位来表示节点的颜色，即红色或黑色。相比之下，红黑树在存储上更加紧凑，占用的内存更少。
  3. 插入和删除操作的性能：由于红黑树的旋转和重新着色操作更少，它的插入和删除操作通常比 AVL 树更快。尽管两者的时间复杂度都是 O (log n)，但红黑树的常数因子更小，因此在实际应用中，红黑树的插入和删除操作通常更快。
  4. 应用场景的适配性：红黑树适用于更广泛的应用场景。由于红黑树的平衡性维护成本较低，它更适用于需要频繁插入和删除操作的场景。而 AVL 树适用于对插入和删除操作的平衡性要求更高的场景。因此，红黑树更常用于实际的数据结构和算法实现中。

# CFS 调度器

TL，DR：

  • 维护一个红黑树作为任务队列；
  • 根据每个任务的虚拟运行时间和优先级值，选择下一个要运行的任务；
  • 运行选中的任务，并更新任务的虚拟运行时间；
  • 重复以上步骤，实现公平调度。

CFS（Completely Fair Scheduler）调度器的核心算法是通过维护一个红黑树来实现公平调度。

CFS 调度器的目标是让所有任务都能够获得公平的 CPU 时间片。为了实现这个目标，CFS 调度器将系统中的所有运行中的任务（包括进程和线程）组织成一个红黑树，称为任务队列。

CFS 调度器使用红黑树来维护任务队列，并且根据每个任务的虚拟运行时间来决定任务的位置。

虚拟运行时间是一个相对于实际运行时间的概念，它表示一个任务应该在 CPU 上运行的时间。

CFS 调度器通过比较每个任务的虚拟运行时间来确定下一个应该运行的任务。具体来说，它选择红黑树中虚拟运行时间最小的任务作为下一个要运行的任务。当一个任务运行时，它的虚拟运行时间会增加，而其他任务的虚拟运行时间则会保持不变。这样，CFS 调度器能够在每次调度时选择最需要 CPU 时间的任务，从而实现公平调度。

CFS 调度器还考虑了优先级的因素。每个任务都有一个优先级值，优先级值越高，任务获取 CPU 时间片的机会越大。在红黑树中，优先级值较高的任务会被放置在较低优先级值的任务之前。

# 表层：

调用 fork 之后，父进程会返回子进程 PID，子进程会返回 0

父进程要调用 wait 来同步子进程的状态，不然会产生孤儿线程（父先退出）或僵尸线程（子先退出）

# 原理：

fork 系统调用触发 0x80 软中断，从用户态到内核态，通过 IDT（中断描述符表）找到系统调用端口

调用具体的系统调用函数，处理之后返回用户态继续执行

C 函数 sys_fork 具体流程：

  1. find_empty_process: 申请新的 PID, 64
  2. copy_process:
    1. 子进程复制父进程的文件描述符（并发写可能有问题）
    2. 部分变量初始化为 0
    3. 复制父进程的页表（写时复制，并未分配实际物理页表）
    4. 设置 GDT 中的 TSS 段（R、权限）和 LDT 段
    5. 返回子进程的 PID

# 写时复制

子与父共享物理内存（只读页），省去大量复制开销

写内存时，发现只读页，触发缺页中断，内核把异常页复制一份
